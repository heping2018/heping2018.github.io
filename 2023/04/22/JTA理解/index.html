JTA

Java Transaction API (JTA) specifies high-level interfaces between a transaction manager and the parties involved in a distributed transaction system: the application, resource manager, and application server.

istributed transaction services in Enterprise Java middleware involves five players:

  • the transaction manager,
  • A transaction manager provides the services and management functions required to support transaction demarcation,transactional resource management,synchronization, and transaction context propagation
  • application server
    • support the application run-time environment which includes transaction state management
  • the resource manager
  • provides the application access to resources
  • The resource manager participates in distributed transactions by implementing a transaction resource interface used by the transaction manager to communicate transaction association, transaction completion and recovery work
  • the application program (事务组件)
  • the communication resource manager (通信资源管理器)
  • Refer to the JTS Specification [2] for more details on interoperability between transaction managers
