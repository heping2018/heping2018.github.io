kubernetes核心概念

Posted on 2022-10-16 Edited on 2023-02-08

docker

Containerd

基本概念

  • master： k8s集群得管理节点，负责管理集群，提供集群得资源数据访问入口。拥有Etcd存储服务，运行Api-server进程，Controller Manager 服务进程以及Scheduler服务进程。
  • node: node是kubernetes集群架构中运行Pod的服务节点，是kubernetes集群操作的单元，用来承载被分配Pod的运行，是Pod的宿主机
  • pod：运行在Node节点上，使用相同的网络的命名空间，IP地址和端口，能够通过localhoast进行通信，Pod是k8s创建，调度和管理的最小单位，它提供了比容器更高层次的抽象，使得部署和管理更加灵活，一个pod可以包含一个容器或者多个容器。
  • label：k8s中label实质是一系列得key/value键值对，其中key与value可自定义。Label可以附加到各种资源对象上，如Node，Pod，Service,Rc等。一个资源对象可以定义任意数量得label，同一个Label也可以被添加到任意数量的资源对象上去，k8s通过Label selector 查询和筛选资源对象。
  • Relication Controller：用来管理Pod得副本，保证集群中存在指定数量得Pod得副本，如果，集群中副本得数量大于指定数量，则会停止指定数量之外得多余容器，反之，则会启动少于指定数量个数容器，保证数量不变，是实现弹性伸缩，动态扩容，滚动升级得核心。
  • Deployment: 内部使用RS来实现目的，Deployment相当与RC得一次升级，其最大得特色为可以随时获取当前Pod部署进度。
  • HPA：Pod得横向自动扩容，也是k8s得一种资源，通过追踪分析RC控制得所有Pod目标得负载变化情况，来确定是否需要针对性得调整Pod副本数量。
  • Service：服务发现，定义了Pod得逻辑集合和访问该集合得策略，是真实服务得抽象，service提供了统一服务访问入口以及服务代理和发现机制，关联多个相同Label得Pod，用户不需要了解后台得pod得如何运行。
  • Volume：Volume是pod中能够被多个容器访问的共享目录，k8s是volume是定义在Pod上，可以被一个或多个Pod的容器挂载到某个目录上。
  • namespace：用于实现多租户的资源隔离，可将集群内部的资源对象分配到不同的Namespace中，形成逻辑的不同项目或用户组，便于不同Namespace在共享使用整个集群的资源的同时还被分别管理。

相关组件

  • kubernetes Api server：作为kubernetes系统的入口，其封装了核心对象的增删改查操作，以Restful API接口方式提供给外部客户和内部组件调用，集群内各个功能模块之间数据交换和通信的中心枢纽
  • kubernetes Schedule：为新建立的Pod进行节点选择合适机器，负责集群的资源调度。
  • kubernetes Controller：负责执行各种控制器。
  • Replication Controller: 管理维护Replication Controller,关联Replication Controller和Pod，保证Relication Controller定义的副本数量和实际运行Pod数量一致。
  • Node Controller：管理维护Node，定期检查Node的健康状态。
  • Namespacce Controller: 维护Namespace，定期清理无效的NameSpace，包括Namespace下的Api对象，比如Pod，service等
  • Service Controller：管理维护service，提供负载以及服务代理。
  • Endpoint Controller：维护Endpoints，关联service和Pod，当Pod发生变化时，实时更新EndPoint。
  • Service Account Controller: 管理维护service Account，为每个NameSpace创建默认的serviceAccount，同时为service Account 创建serviceAccount secret。
  • Persistent volume Controller：管理维护Persistent volume和Persistent volume claim，为新的Persistent volume claim分配Persistent volume进行绑定，为释放的Persistent volume执行清理回收。
  • Daemon Set Controller：维护Daemon set，负责创建Deamon pod，保证指定的Node上正常运行Daemon Pod。
  • Deployment Controller：维护Deployment，关联Deployment和Relication Controller，保证运行指定数量的Pod，当Deployment更新时，控制实现Replicaion Controller和Pod更新。
  • Job Controller：维护Job，为Job创建一次性任务Pod，保证完成Job指定完成的任务数目。
  • Pod Autoscaler Controller：实现Pod的自动伸缩，定时获取监控数据，进行策略匹配，当满足执行pod的伸缩策略。
